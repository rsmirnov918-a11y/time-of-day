# Time of Day — личный планировщик дня и недели

## Что это
Одностраничное веб-приложение (один файл `index.html`, ванильный JS + встроенные CSS, без сборщика и фреймворков). Личный планировщик дня/недели с drag-and-drop. Открывается в браузере, хостится на GitHub Pages, синхронизируется между устройствами через Firebase.

## Стек и архитектура
- **Один файл** `index.html`: HTML + `<style>` + `<script type="module">`. Никакого npm/сборщика.
- **Drag** реализован на Pointer Events вручную (не SortableJS), чтобы работало на тач и мышь без CDN-зависимостей.
- **Firebase** подключается через ES-модули с CDN gstatic (firebase-app, firebase-auth, firebase-firestore).
- **Хранение** вынесено в отдельный слой load/save: localStorage (офлайн-кэш) + Firestore (облако). Так слой можно менять, не трогая логику UI.

## Главная механика — «ВРЕМЯ-ТРЕК» (НЕ ЛОМАТЬ при доработках)
- День — это **фиксированный временной трек** (окно по умолчанию 07:00–23:00, поля «Старт»/«до» редактируются, не захардкожены). Трек виден всегда на всё окно, даже если задач мало/нет.
- У каждой задачи СВОЁ **время старта** `task.start` (минуты от полуночи) и длительность `dur`. Это РЕАЛЬНОЕ хранимое поле (не вычисляется из порядка — старая модель «встык» заменена).
- **Свободное время = пустые промежутки на треке** (рисуются как `.slot` с подписью «свободно … · + задача»; тап = добавить задачу на это время). Отдельных блоков «Буфер»/«Свободное» в базе нет.
- Высота блока пропорциональна длительности (1 час = 60px = 1мин/1px). Видимая высота на ~6px меньше слота — косметический зазор между блоками (`visH`), на время НЕ влияет.
- **Наложений нет**: при любом изменении раскладка каскадно раздвигается (`computeLayout` сортирует по start и не даёт блоку начаться раньше конца предыдущего). При перетаскивании/правке активный блок — «якорь» (`placeAnchored`), соседние раздвигаются вокруг него (вверх до якоря, вниз после). После изменения вызывается `normalize()` — пишет разрешённые start обратно и сортирует.
- Слева — часовая шкала (метки 07:00…23:00). Трек тянется до `max(окно, конец последней задачи)`.
- Все времена/длительности кратны 5 минутам, минимум 10 минут.

## Взаимодействие
- Тащить блок (на телефоне — за ручку ⠿ слева, она `touch-action:none`; тело блока на тач скроллит трек и открывает редактор по тапу) = **двигать по времени**; соседние раздвигаются, наложений нет.
- Тянуть за нижний край = менять длительность (ресайз, шаг 5 мин); нижние блоки раздвигаются.
- Тап (порог ~6px) = редактор: название, категория, **время начала** (двигает блок туда), длительность (±5/±15), «На каких днях» (чипы Пн–Вс + «Все дни» — создать копию задачи на других днях, связь по `gid`), «Удалить» (этот день) и «Со всех дней» (вся группа `gid`).
- Короткие блоки (<30 мин): полоса ресайза скрыта, тап открывает редактор (длительность меняется степперами).
- «+ Задача» = добавить блок (60 мин, «Свободное») в конец дня и открыть редактор. Тап по пустому промежутку — добавить на это время.
- Undo/redo: кнопки ↶ ↷ в шапке + Ctrl+Z / Ctrl+Y (Ctrl+Shift+Z), быстрые правки коалесятся в один шаг; приход из облака сбрасывает историю.
- Вкладки дней Пн–Вс; сегодняшний подсвечен, на нём горизонтальная линия «сейчас» (обновляется раз в 30с).
- `gid` — групповой id «один и тот же блок на разных днях». Копии независимы (правки одного дня не синкаются на другие); связь по `gid` нужна для чипов дней и «удалить со всех дней».

## Сводки
- Под вкладками для текущего дня: «Сессия Xч · Свободно ≈Yч».
- Где-то компактно — недельные суммы (Покер-сессии и Теория за неделю).
- Все суммы пересчитываются в реальном времени.

## Хранение и синхронизация
- **localStorage**: автосохранение при каждом изменении (debounce ~250мс), офлайн-кэш.
- **Firestore**: документ `schedules/{uid}`, формат `{ state: <объект состояния>, updatedAt: serverTimestamp() }`.
- **Auth**: вход через Google (signInWithPopup). Без входа приложение работает локально на localStorage.
- При входе: onSnapshot на документ пользователя → изменения из облака применяются к UI в реальном времени (это и есть синк телефон↔ноут). Если в облаке пусто, а локально есть данные — миграция local→cloud (только при подтверждённом сервером снимке, не из кэша, чтобы не затереть облако). Если документ в облаке есть — он источник правды.
- Запись в Firestore: setDoc с merge, debounce ~600мс. Если не авторизован — в облако не писать.
- Оффлайн: enableIndexedDbPersistence в try/catch. Защита от «эха»: не отправлять обратно изменение, прилетевшее из onSnapshot.
- Экспорт/импорт JSON и сброс к шаблону — сохранять рабочими.

## Критерии приёмки (проверять после изменений)
1. На телефоне палец таскает блок (за ручку ⠿), страница не скроллится.
2. **Наложений нет ни при каких действиях** (drag/ресайз/правка времени → блоки раздвигаются, overlaps:0). Свободные промежутки между блоками — это норма (свободное время).
3. Ресайз за нижний край меняет длительность; нижние блоки раздвигаются.
4. Тап открывает редактор; правки (в т.ч. «время начала» двигает блок) сразу видны и сохраняются.
5. Перезагрузка сохраняет всё (localStorage + облако после входа). Старые данные без `task.start` мигрируют (время проставляется встык от старта дня).
6. Переключение дней; линия «сейчас» на сегодняшнем дне; трек всегда на всё окно 07:00–23:00.
7. Экспорт/импорт/сброс работают. Undo/redo (кнопки + Ctrl+Z/Y).
8. «На каких днях» создаёт копии на выбранных днях; «Со всех дней» удаляет группу `gid`.
9. Вход на двух устройствах одним аккаунтом → синк без перезагрузки; нет бесконечных циклов записи от onSnapshot.
