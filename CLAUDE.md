# Time of Day — личный планировщик дня и недели

## Что это
Одностраничное веб-приложение (один файл `index.html`, ванильный JS + встроенные CSS, без сборщика и фреймворков). Личный планировщик дня/недели с drag-and-drop. Открывается в браузере, хостится на GitHub Pages, синхронизируется между устройствами через Firebase.

## Стек и архитектура
- **Один файл** `index.html`: HTML + `<style>` + `<script type="module">`. Никакого npm/сборщика.
- **Drag** реализован на Pointer Events вручную (не SortableJS), чтобы работало на тач и мышь без CDN-зависимостей.
- **Firebase** подключается через ES-модули с CDN gstatic (firebase-app, firebase-auth, firebase-firestore).
- **Хранение** вынесено в отдельный слой load/save: localStorage (офлайн-кэш) + Firestore (облако). Так слой можно менять, не трогая логику UI.

## Главная механика (НЕ ЛОМАТЬ при доработках)
- День — это **упорядоченный список** задач, у каждой задачи есть длительность.
- Абсолютное время задачи **вычисляется** (computeLayout): старт дня + сумма длительностей всех задач выше по списку. Время НЕ хранится.
- Из-за этого **наложения блоков невозможны в принципе** — раскладка всегда выводится из порядка. Это ключевое требование, в ранних версиях блоки накладывались — так быть не должно.
- Высота блока пропорциональна длительности (1 час = 60px), блоки идут стык в стык.
- Перетаскивание = пересортировка списка (drag-to-reorder): соседи раздвигаются, точка вставки по центру перетаскиваемого блока. На отпускании список переупорядочивается, времена пересчитываются.
- Свободное время и перерывы — обычные задачи (категории «Свободное»/«Перерыв»), чтобы таймлайн попадал в реальные часы.
- Старт дня по умолчанию 07:00 (редактируется в поле «Старт»), не должен быть захардкожен.
- Все времена/длительности кратны 5 минутам, минимум 10 минут.

## Взаимодействие
- Тащить тело блока (на телефоне — за ручку слева) = двигать по списку.
- Тянуть за нижний край = менять длительность (ресайз, шаг 5 мин).
- Тап (без перетаскивания, порог ~6px) = открыть редактор: название, категория, длительность (±5/±15), удалить.
- «+ Задача» = добавить блок (60 мин, категория «Свободное»), сразу открыть редактор.
- Вкладки дней Пн–Вс; сегодняшний подсвечен, на нём горизонтальная линия «сейчас» (обновляется раз в 30с).

## Сводки
- Под вкладками для текущего дня: «Сессия Xч · Свободно ≈Yч».
- Где-то компактно — недельные суммы (Покер-сессии и Теория за неделю).
- Все суммы пересчитываются в реальном времени.

## Хранение и синхронизация
- **localStorage**: автосохранение при каждом изменении (debounce ~250мс), офлайн-кэш.
- **Firestore**: документ `schedules/{uid}`, формат `{ state: <объект состояния>, updatedAt: serverTimestamp() }`.
- **Auth**: вход через Google (signInWithPopup). Без входа приложение работает локально на localStorage.
- При входе: onSnapshot на документ пользователя → изменения из облака применяются к UI в реальном времени (это и есть синк телефон↔ноут). Если в облаке пусто, а локально есть данные — миграция local→cloud (только при подтверждённом сервером снимке, не из кэша, чтобы не затереть облако). Если документ в облаке есть — он источник правды.
- Запись в Firestore: setDoc с merge, debounce ~600мс. Если не авторизован — в облако не писать.
- Оффлайн: enableIndexedDbPersistence в try/catch. Защита от «эха»: не отправлять обратно изменение, прилетевшее из onSnapshot.
- Экспорт/импорт JSON и сброс к шаблону — сохранять рабочими.

## Известные задачи / возможные доработки
- **Баг со временем** (надо проверить): где-то итог дня не сходится (например воскресенье заканчивалось в 21:55 вместо 22:40 при старте 07:00). Причина — суммы длительностей в дне не дают ожидаемых 940 мин. Проверять computeLayout и длительности блоков в seed.
- Индикатор статуса синка в шапке («синхронизировано / офлайн»).
- «Закреплённое время» для задачи: опция, при которой перед задачей авто-добавляется свободный зазор, чтобы она начиналась в заданный час (например сессия строго в 11:00).
- Копирование дня на другой день. Тёмная тема.

## Критерии приёмки (проверять после изменений)
1. На телефоне палец таскает блок, страница не скроллится.
2. Наложений нет ни при каких действиях (overlaps:0), блоки стык в стык (gaps:0).
3. Ресайз за нижний край меняет длительность, время пересчитывается.
4. Тап открывает редактор; правки сразу видны и сохраняются.
5. Перезагрузка сохраняет всё (localStorage + облако после входа).
6. Переключение дней; линия «сейчас» на сегодняшнем дне.
7. Экспорт/импорт/сброс работают.
8. Вход на двух устройствах одним аккаунтом → изменения синхронизируются без перезагрузки.
9. Нет бесконечных циклов записи от onSnapshot.
